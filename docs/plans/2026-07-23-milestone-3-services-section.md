# Milestone 3 — Services Section Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a services section presenting the 3 Hunter.AI offers (AI agents, site creation, site redesign) as distinct cards below the hero, with outcome-focused copy (no automation-tooling leaks) and a scroll-triggered entrance consistent with the hero's motion language.

**Architecture:** One new component, `components/sections/services.tsx` (client component — needs framer-motion `whileInView` + `prefers-reduced-motion` detection, same pattern as `components/sections/hero.tsx`), rendered in `app/page.tsx` right after `<Hero />`. Uses the shadcn `Card` primitive (added fresh this milestone) and `lucide-react` icons (already a dependency). No changes to `WhatsAppCta`, `lib/whatsapp.ts`, `SiteHeader`, or `ParticleField`.

**Tech Stack:** Next.js App Router, React, TypeScript, Tailwind CSS v4 (CSS-based theme in `app/globals.css`), shadcn/ui (`Card`), lucide-react, framer-motion (existing dependency).

**Design reference:** `docs/plans/2026-07-23-milestone-3-services-section-design.md` — read in full before starting; it has the exact copy, layout, and motion spec.

**Note on process:** No test runner in this project (confirmed convention from Milestones 1–2). Verification is manual: `npm run dev` + browser check, `npm run build` at the end.

## Global Constraints (binding — copy into every task reviewer dispatch)

- Colors: only `bg-ink` (`#0A0F0D`), `text-signal`/`bg-signal`/`border-signal` (`#2EE6A0`), `text-amber`/`bg-amber` (`#FFB23E`), and the existing zinc/white neutrals already used elsewhere. No new colors, no gradients.
- No mention of "n8n" or any automation-tool name anywhere in copy, comments, or commit messages.
- No contact form, email capture, per-card links/CTAs, or backend/database call of any kind — the only interactive element on the page stays the existing `WhatsAppCta` (hero only, at this milestone).
- All scroll-triggered entrance animation MUST be skipped (render final state immediately) when `window.matchMedia("(prefers-reduced-motion: reduce)").matches` is `true` — same hard requirement as the hero.
- 3 cards only, equal visual weight (no "featured" card), no carousel/horizontal scroll.
- Reuse the shadcn `Card` primitive once added — don't hand-roll a competing card div structure.

### Task 3: Build the `Services` section component

**Mandatory tooling:** the `frontend-design` skill (visual quality/layout execution) and the `motion-design` skill (entrance stagger timing/easing) MUST be used while implementing this task.

**Files:**
- Create: `components/sections/services.tsx`
- Modify: `app/page.tsx`

**Step 1:** Read `docs/plans/2026-07-23-milestone-3-content-spec.md` (Task 2's output) first — it has the exact, final copy strings to use verbatim. Do not re-draft copy in this task.

**Step 2:** Build a client component (`"use client"`) with the structure from the design doc:

- `<section id="servicos" className="bg-ink py-24 sm:py-32">` containing a `<div className="mx-auto max-w-6xl px-6">` wrapper.
- Header block (left-aligned, `max-w-2xl`):
  - Eyebrow pill (same visual treatment as the hero's: `inline-flex bg-white/5 backdrop-blur-sm border border-white/10 rounded-full px-4 py-1.5 font-mono text-xs uppercase tracking-widest text-amber`)
  - `h2` (`font-display text-3xl font-semibold tracking-tight text-zinc-50 sm:text-4xl`)
  - `p` (`font-body text-base text-zinc-400 sm:text-lg mt-4`)
  - (all three: exact text from the content spec)
- Grid below the header (`grid grid-cols-1 gap-6 sm:grid-cols-3 mt-16`), 3 `Card`s, one per service (exact copy from the content spec — `MessageCircle` / `Globe` / `RefreshCw` icons from `lucide-react`, each in a `bg-signal/10 border border-signal/20` square accent box with `text-signal` icon color). Each card shows: icon box → numbered index (`01`/`02`/`03`, `font-mono text-xs text-zinc-500`) → `CardTitle` → `CardDescription`/body text. Add `hover:border-signal/40 hover:-translate-y-1 transition` on each `Card` for the subtle hover lift from the design doc (no click behavior — cards aren't links).
- Embed the JSON-LD `Service` structured data from the content spec via `<script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(serviceSchema) }} />` inside the section (define `serviceSchema` as a module-level `const` object above the component, verbatim from the content spec).

**Step 3:** **Reduced-motion + entrance animation:** reuse the exact `prefers-reduced-motion` detection pattern already in `components/sections/hero.tsx` (read into state on mount, subscribe to the `change` event). Wrap the header block and the card grid in framer-motion (`motion.div` with `variants`, `initial="hidden"`, `whileInView="visible"`, `viewport={{ once: true, amount: 0.3 }}`), staggering: header fades/slides in first, then the 3 cards with a small `staggerChildren` left-to-right — use the `motion-design` skill to pick the stagger delay/easing so it reads as the same system as the hero's mount animation, not a different feel. When `prefersReducedMotion` is `true`, set `initial={false}` so everything renders in its final state with no animation.

**Step 4:** Wire into `app/page.tsx`:
```tsx
import { Hero } from "@/components/sections/hero";
import { Services } from "@/components/sections/services";

export default function Home() {
  return (
    <>
      <Hero />
      <Services />
    </>
  );
}
```

**Step 5:** Run `npm run dev` and check in-browser:
- Scrolling from the hero into the services section triggers the fade/stagger entrance once (not on every scroll up/down past the threshold — `once: true` handles this, confirm visually).
- All 3 cards render with correct icon, number, title, and body copy (matching the content spec exactly); equal width/height alignment on desktop (1280px+) and clean single-column stack on mobile (360px), no horizontal scroll.
- Hover each card on desktop: subtle border/lift transition, no jank.
- Toggle `prefers-reduced-motion: reduce` (via browser devtools rendering emulation) and reload/re-scroll: header and cards appear immediately in final position, no animation.
- View page source / inspect the DOM: the `application/ld+json` script tag is present with valid JSON (no trailing commas/syntax errors — `JSON.parse` it manually if unsure).
- Confirm no "n8n"/automation-tool wording anywhere in the rendered section.
- `npm run build`: production build is clean (no type errors, no lint errors).

**Step 6:** Commit:
```bash
git add components/sections/services.tsx app/page.tsx
git commit -m "feat: add Services section"
```
